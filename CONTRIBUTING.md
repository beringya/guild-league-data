# 贡献指南

## 分支规范

- `main`：稳定版本。
- `develop`：日常集成。
- `feature/*`：功能开发。
- `fix/*`：缺陷修复。

## 提交规范

建议使用：

```text
feat: 新功能
fix: 修复问题
docs: 文档更新
style: 样式调整
refactor: 重构
test: 测试
chore: 构建或杂项
```

## 合并请求

提交 MR/PR 前请确认：

- 变更范围与需求一致。
- 不提交 `.env`、数据库文件、上传文件和备份文件。
- 说明是否执行过构建、测试或 Docker 验证。
- 涉及导入/评分逻辑时补充样例或测试。
