# 完整开发计划

## 阶段 1：工程骨架与部署

- 建立 `backend/`、`frontend/`、`deployment/`、`data/`、`backups/` 目录。
- 后端采用 FastAPI + SQLAlchemy + SQLite，启动时创建表结构、启用 WAL、初始化设置和管理员。
- 前端采用 React + TypeScript + Vite，复用设计文档中的品牌、背景和图标资源。
- 提供 Dockerfile、docker-compose、`.env.example` 和入口脚本。

## 阶段 2：登录与安全

- 实现 `admin` 首次随机密码、Argon2id 哈希、强制改密。
- 实现会话 Cookie、CSRF Token、退出、修改密码、登录失败限流。
- 所有业务 API 默认需要认证，健康检查和登录除外。

## 阶段 3：CSV 导入与入库

- 支持 UTF-8 BOM、UTF-8、GB18030。
- 清洗重复表头、空行、数字逗号、字段别名。
- 校验两个帮会、十三个已确认职业、必填字段和非法数字。
- 预览阶段返回帮会人数、职业、分团、警告、错误、SHA-256 和推断比赛时间。
- 确认导入后保存比赛、帮会、玩家、原始统计和导入日志，防止重复导入。

## 阶段 4：分析引擎

- 实现 KDA、参团率、伤害占比、职业区间标准化。
- 首次无范围时按同场双方同职业样本生成并发布冻结范围。
- 按每职业六维规则计算综合分、单项贡献、百分位和可解释 JSON。
- 实现本帮/对手/双方排名、同职业排名、分团排名和并列排序。
- 实现帮会总量/人均/职业人均对比、分团对比、团内 TOP3 和规则化结论。
- 实现多场总榜，默认按场均综合分排序，支持最低参赛场次。

## 阶段 5：前端页面

- 登录页、首页概览、个人排名、玩家详情、团内 TOP3、对手对比、团队对比、导入、历史、设置。
- 使用真实 API 数据流，空数据时提供导入入口。
- 使用 ECharts 展示柱状图、雷达图和趋势图。
- 使用设计文档粉色主题、固定侧栏、顶部操作栏、卡片和表格样式。

## 阶段 6：测试与验收

- 覆盖 CSV 清洗、重复表头、样例人数、KDA、参团率、范围建议和综合分。
- 提供静态文件级检查说明。
- 因当前本地没有部署环境，本轮不执行编译运行；后续可在 Docker 环境中执行端到端验收。
