# 完整开发计划

> 当前计划已按用户要求调整为参考 `sub2api` 的技术栈与安装体验：后端 Go + Gin + Ent，前端 Vue 3 + Vite + TailwindCSS，数据库 PostgreSQL 15+，缓存/会话 Redis 7+。  
> 本地当前没有部署环境，本轮只做源码、文档、脚本和静态文件级交付，不执行编译运行。

## 阶段 1：工程骨架与技术栈切换

- 已归档旧版 FastAPI + React 实现到 `legacy/fastapi-react/`，仅保留为参考。
- 已重建 `backend/` 为 Go 服务：`cmd/server` 入口、`internal/` 分层、`ent/schema` 数据模型、PostgreSQL migration。
- 已重建 `frontend/` 为 Vue 3 + TypeScript + Vite + TailwindCSS 应用，复用设计文档中的品牌、背景和图标资源。
- 已保持 `deployment/`、`data/`、`backups/`、`scripts/` 目录结构，部署、上传、备份数据与代码分离。

## 阶段 2：数据库、缓存与独立 Docker 部署

- 已默认使用 PostgreSQL 15+，数据表覆盖用户、会话、帮会、比赛、玩家、统计、评分规则、范围版本、头像、导入日志和设置。
- 已使用 Ent schema 作为 Go 侧领域模型与后续代码生成入口，同时保留 SQL migration。
- 已使用 Redis 7+ 存储会话、登录失败限流和短期导入预览缓存。
- `docker-compose.yml` 已默认启动 `app`、`postgres`、`redis` 三个独立服务，使用独立 project、network、volume 和容器名。
- 默认宿主机端口为 `18080`，可通过 `.env` 或安装脚本覆盖。

## 阶段 3：登录与安全

- 已实现首次启动自动创建管理员账号 `admin`，随机生成临时密码，仅在首次初始化日志中输出一次。
- 已使用 Argon2id 保存密码，首次登录后强制修改密码。
- 已实现 HttpOnly SameSite Cookie + Redis 会话；业务写操作校验 CSRF Token。
- 已实现登录失败按用户名和来源 IP 限流；修改密码后撤销其他会话。
- 已实现除健康检查与登录接口外，业务 API 默认要求认证。

## 阶段 4：CSV 导入与入库

- 已支持 UTF-8 BOM、UTF-8、GB18030 编码解析。
- 已清洗重复表头、空行、数字逗号、字段别名和空数字。
- 已校验两个帮会、十三个已确认职业、必填字段、非法数字和未来未配置职业。
- 预览阶段已返回帮会人数、职业分布、分团统计、警告、错误、SHA-256 和推断比赛时间，并缓存到 Redis。
- 确认导入后已写入比赛、帮会、玩家、原始统计、导入日志和评分结果，按 SHA-256 防止重复导入。

## 阶段 5：分析引擎

- 已实现 KDA、参团率、玩家伤害占比、建筑伤害占比、职业区间标准化和转化率。
- 已实现按同场双方同职业样本生成范围建议：`n >= 20` 使用 P5/P95，`3 <= n < 20` 使用最小/最大值加安全余量，`n < 3` 给出高风险提醒。
- 已按每职业六维规则计算综合分、单项贡献、百分位和可解释 JSON。
- 已实现本帮/对手/双方排名、同职业排名、分团排名、并列排序和稳定排序。
- 已实现帮会总量、人均、职业对比、分团对比、团内 TOP3 和规则化结论。
- 已实现多场历史总榜，默认按场均综合分排序，支持最低参赛场次、职业、帮会和玩家搜索。

## 阶段 6：前端页面

- 已覆盖登录、首页概览、个人排名、多场总榜、玩家详情、团内 TOP3、对手帮会对比、团队数据对比、数据导入、历史记录、设置。
- 已使用 Vue Router 管理页面，使用 Pinia 管理登录态。
- 已使用 ECharts 展示柱状图和雷达图，页面消费真实 API 数据。
- 已使用 TailwindCSS 和自定义主题承接粉色后台风格、固定侧栏、顶部操作栏、表格、筛选器和状态标签。
- 空数据时优先展示导入入口，不做营销落地页。

## 阶段 7：设置、规则与头像

- 设置页管理默认本帮会、多场榜最低场次、优势判断阈值、自动备份和管理员密码。
- 支持职业六维指标、上下限、方向、权重和版本发布；历史比赛不会静默变更，只有显式重新分析才更新派生结果。
- 支持生成范围建议、发布冻结范围版本和查看样本风险提醒。
- 玩家头像使用稳定随机头像兜底，支持上传替换；职业默认头像作为第二级兜底。

## 阶段 8：自动化安装与打包

- 参考 `sub2api` 的脚本安装体验，提供可离线解包后执行的 `scripts/install.sh`。
- 安装脚本负责架构/依赖检查、生成 `.env`、随机生成 `POSTGRES_PASSWORD` 与 `SESSION_SECRET`、创建 systemd 服务、启动 Docker Compose、输出访问地址和首次密码查看命令。
- 提供 `scripts/uninstall.sh`，默认保留数据，显式 `REMOVE_DATA=true` 才删除安装目录和 Docker 卷。
- 提供 `scripts/package.sh`，生成包含源码、前端资源、Docker、Compose、脚本、Gitee 模板和设计文档摘要的发布包。

## 阶段 9：Gitee 与交付规范

- 补齐并维护 `.gitignore`、`.dockerignore`、`LICENSE`、`CONTRIBUTING.md`、`.gitee/ISSUE_TEMPLATE.zh-CN.md` 和 `.gitee/PULL_REQUEST_TEMPLATE.zh-CN.md`。
- README 明确技术栈、数据库选择、一键安装、Docker 独立部署和本地未编译运行说明。
- 交付包不包含 `.env`、数据库文件、上传文件、备份文件、`node_modules`、Go 编译产物或临时日志。

## 阶段 10：验证与验收

- 源码级检查：目录结构、配置项、Docker Compose 服务、安装脚本变量、API 路由、前端页面入口和静态资源引用。
- 后端测试计划：CSV 清洗、重复表头、样例人数、KDA、参团率、范围建议、综合分、权限与会话。
- 前端测试计划：登录态、导入流程、空数据状态、榜单筛选、图表渲染、响应式布局。
- 因当前本地没有部署环境，本轮不执行 `go test`、`npm run build` 或 Docker 启动；后续在部署环境中执行端到端验收。
